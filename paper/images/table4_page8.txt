## Pertinent Paper Text
Section V-B (Ablation Study): "We ablate four design choices: (1) Mode Dropout — randomly dropping the mode condition during training to prevent over-reliance; (2) Selector Side Task — auxiliary IL loss on ego trajectory prediction from the mode selector; (3) Ego-history Dropout — randomly masking ego history to improve robustness; (4) Backbone Sharing — sharing the visual backbone between policy and value function."

"For RL, the best configuration uses Mode Dropout + Selector Side Task (without Ego-history Dropout or Backbone Sharing), achieving CLS-NR=94.07."

"Ego-history Dropout hurts RL performance (89.51 vs 94.07), suggesting the policy needs ego history for RL credit assignment."

## Key Implementation Notes
- **RL best config**: Mode Dropout=✓, Selector Side Task=✓, Ego-history Dropout=✗, Backbone Sharing=✗ → CLS-NR=94.07
- **IL best config**: all four=✓ → CLS-NR=93.41
- Mode Dropout is critical for both IL and RL
- Ego-history Dropout helps IL but hurts RL
- Backbone Sharing helps IL but hurts RL
- Loss Generator values for RL are much larger (540–2254) than IL (147–174) — different scale
